## Visual Test Mode Checklist

- [x] Task A — `lua/headless/runner.lua` — In `runner.setup()`, assign `runner._visual_ctx = ctx` immediately before the `return ctx` line. This gives `love.draw()` a stable reference to the most-recently-configured context. No other logic changes.

- [x] Task B — `lua/headless/runner.lua` — In `runner.tick()`, add a coroutine yield at the end of the `for _ = 1, n` loop body: after `ctx.sm:update(dt)`, add `if coroutine.running() then coroutine.yield() end`. This suspends the test coroutine once per tick so `love.draw()` can fire between frames. When running headless, `coroutine.running()` returns nil and the branch is never taken.

- [x] Task C — `main.lua` — Add a `--visual` parse block at the top of the file, mirroring the existing `--headless` block (lines 1–15). Inside a `do … end` block, iterate `arg` to detect `--visual` and capture the optional test-file path using the same sign-check pattern (`v:sub(1,1) ~= "-"`). When visual mode is active: (1) do **not** load stubs; (2) define `love.load()` to call `love.graphics.setDefaultFilter("nearest","nearest")`, create the 1280×720 canvas, set `_G.runner = require("lua/headless/runner")`, and wrap `dofile(test_file)` in a coroutine stored as `test_co`; (3) define `love.update(dt)` to resume `test_co` and, if the coroutine is dead afterward, print `"PASS"` and `love.event.quit(0)` on success or `"FAIL: " .. err` and `love.event.quit(1)` on failure; (4) define `love.draw()` to read `runner._visual_ctx` and, when non-nil, render through the canvas/scale letterbox pipeline identical to the normal path (`setCanvas`, `clear`, `ctx.sm:draw()`, `setCanvas()`, compute scale/offset, `love.graphics.draw`); (5) define `love.keypressed(key)` to quit on escape; (6) `return` so the normal `love.load` block below is not executed. The existing `--headless` block and the normal `love.*` definitions below must remain completely unchanged.

  **Dependency note:** Task C depends on Tasks A and B being complete first, because `love.draw()` reads `runner._visual_ctx` and `love.update()` relies on the coroutine yield inside `runner.tick()`.
