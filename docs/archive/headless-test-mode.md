## Headless Test Mode Checklist

- [x] Task 1 — `lua/test/love_stubs.lua` *(new file)* — Create the Love2D stub module. Provide stub implementations of every API touched at require/construct time: `love.graphics.newImage(path)` returning a mock with `getWidth()→1` and `getHeight()→1`; `love.graphics.newShader(src)` returning a mock with no-op `.send()` and `.hasUniform()→false`; `love.graphics.newFont(...)` returning a mock with `.getHeight()→12` and `.getWidth()→8`; `love.graphics.newCanvas(...)` returning a mock with no-op `.setFilter()`; `love.filesystem.getInfo(path)→nil`; `love.keyboard.isDown(key)→false`. The module must write stubs into the live `love` global table and return nothing (or a small install function called immediately). Everything else in the feature depends on this file existing.

- [x] Task 2 — `conf.lua` — Add test-mode detection and module disabling. Add a local `is_test_mode()` helper at the top of the file that iterates `arg` looking for `"--test"`. Inside `love.conf(t)`, when `is_test_mode()` is true, set `t.window = false` and set `t.modules.graphics`, `t.modules.window`, `t.modules.audio`, and `t.modules.sound` all to `false`. The normal non-test path must remain identical. Depends on nothing else but must be done before Task 3 so that main.lua can rely on modules already being disabled.

- [x] Task 3 — `main.lua` — Add test-mode bootstrap and `love.run` override. Add the same `is_test_mode()` helper (or require a shared utility if preferred). When test mode is active: (1) call `require("lua/test/love_stubs")` before any game module is required, installing stubs into the `love` global; (2) override `love.run` so that instead of the normal game loop it calls `require("lua/test/runner").run()` and returns. The existing `love.load`, `love.update`, `love.draw`, and `love.keypressed` bodies must be left completely untouched. Depends on Task 1 (stubs) and Task 2 (conf patch) being in place first.

- [x] Task 4 — `lua/test/headless_input.lua` *(new file)* — Create the HeadlessInput class. It must expose the same public interface as `lua/core/input.lua`: `HeadlessInput.new()`, `input:update()`, `input:is_down(action)→bool`, `input:pressed(action)→bool`. Add two test-driving methods: `input:set_down(action, bool)` which holds or releases an action, and `input:press(action)` which queues a single-frame press that is automatically cleared by the next `update()` call. No dependency on `love.keyboard`. This task is independent of Task 5 and can be done in parallel with it.

- [x] Task 5 — `lua/test/t.lua` *(new file)* — Create the minimal assertion library. Implement four functions under a module table `T`: `T.assert(cond, msg)` which raises an error string if `cond` is falsy; `T.eq(a, b, msg)` which raises if `a ~= b`; `T.approx(a, b, eps, msg)` which raises if `math.abs(a-b) > eps`; `T.err(fn, msg)` which raises if calling `fn()` does NOT throw. All failures must raise a string error (not call `love.event.quit`) so the runner can catch them via `pcall`. This task is independent of Task 4 and can be done in parallel with it.

- [x] Task 6 — `lua/test/runner.lua` *(new file)* — Create the test runner. Implement a `runner.run()` function that: scans `tests/unit/` and `tests/integration/` for files matching `*_test.lua`; for each file, calls `pcall(require, path)` and records PASS or FAIL along with the error message on failure; prints `PASS <name>` or `FAIL <name>: <error>` per test; prints a summary line (e.g. `5/6 passed`) after all tests; calls `love.event.quit(0)` if all passed and `love.event.quit(1)` if any failed. Depends on Tasks 4 and 5 existing so that test files requiring them do not error during the scan.

- [x] Task 7 — `tests/unit/plant_test.lua` *(new file, new dir)* — Create the Plant unit test. Require `lua/game/items/plant` and `lua/test/t`. Construct a `Plant.new(1)` and use `T.eq` to verify: `p.stage == 1` on construction; `p.ready == false` on construction; `p.ready == true` after calling `p:update(p._cooldown.duration + 0.01)` (i.e. enough dt to fire the cooldown); `p.stage == 2` after calling `p:water()`. This task depends on all prior tasks (1–6) being complete.

- [x] Task 8 — `tests/integration/currency_test.lua` *(new file, new dir)* — Create the currency integration test. Require `lua/game/game_state`, `lua/game/scenes/store_scene`, `lua/test/headless_input`, and `lua/test/t`. Construct a `GameState.new()`, a `HeadlessInput.new()`, a minimal scene-manager stub (`{ switch = function() end }`), and a `StoreScene.new(gs, input, sm)`. Drive `scene:update(dt)` to simulate a completed customer sale, then use `T.assert` or `T.eq` to verify that `gs.currency` changed by the expected amount. Never call `scene:draw()`. This task depends on all prior tasks (1–6) being complete, and is independent of Task 7.
